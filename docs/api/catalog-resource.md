<span data-ttu-id="836ba-422">Katalog zamanına göre dizini oluşturulmuş bir yalnızca ekleme yapılabilen bir veri yapısı olduğundan, istemci depolamanız gerekir bir **imleç** ne istemcinin belirli bir noktaya kadar temsil eden katalog öğeleri yerel olarak işledi.</span><span class="sxs-lookup"><span data-stu-id="836ba-422">Because the catalog is an append-only data structure indexed by time, the client should store a **cursor** locally, representing up to what point in time the client has processed catalog items.</span></span> <span data-ttu-id="836ba-423">Bu imleç değeri hiçbir zaman istemci makine düzende oluşturulacağını unutmayın.</span><span class="sxs-lookup"><span data-stu-id="836ba-423">Note that this cursor value should never be generated using the client's machine clock.</span></span> <span data-ttu-id="836ba-424">Bunun yerine, değeri bir katalog nesnenin gelmesi gerektiğini `commitTimestamp` değeri.</span><span class="sxs-lookup"><span data-stu-id="836ba-424">Instead, the value should come from a catalog object's `commitTimestamp` value.</span></span>

<span data-ttu-id="836ba-425">İstemci paket kaynağı yeni olayları işlemek istediği her durumda ihtiyaç yalnızca katalog sorgu yürütme zaman damgasına sahip tüm katalog öğeleri için kendi saklı imleç büyüktür.</span><span class="sxs-lookup"><span data-stu-id="836ba-425">Every time the client wants to process new events on the package source, it need only query the catalog for all catalog items with a commit timestamp greater than its stored cursor.</span></span> <span data-ttu-id="836ba-426">İstemci başarıyla tüm yeni katalog öğelerini işledikten sonra katalog öğelerini yalnızca yeni imleç değer olarak işlenen en son işleme zaman damgası kaydeder.</span><span class="sxs-lookup"><span data-stu-id="836ba-426">After the client successfully processes all new catalog items, it records the latest commit timestamp of catalog items just processed as its new cursor value.</span></span>

<span data-ttu-id="836ba-428">Ayrıca, istemci, imlecin kayıtlı işleme zaman damgası önce eski olayları yeniden işlemek hiçbir zaman vardır.</span><span class="sxs-lookup"><span data-stu-id="836ba-428">Additionally, the client never has to reprocess old events prior to the cursor's recorded commit timestamp.</span></span>

<span data-ttu-id="836ba-429">Bu güçlü kavramı imleçler, birçok nuget.org arka plan işleri için kullanılır ve V3 API güncel tutmak için kullanılır.</span><span class="sxs-lookup"><span data-stu-id="836ba-429">This powerful concept of cursors is used for many of nuget.org background jobs and is used to keep the V3 API itself up-to-date.</span></span>
